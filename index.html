不求甚解，但求可行：Max上安装Hexo小记

曾几何时听某位阿里高层的分享，各种纵横捭阖，但有一句话记得清楚：他说不喜欢“黑盒子”。他说别人跟他汇报他遇到“黑盒子”就要打破砂锅问到底。

如果你写过程序，做过debug，会在跟进代码的时候选择“跳进去”还是“跳过去”。代码一层一层封装，一层一层透明，是为好代码。

好了，作为一篇技术博客里的技术文章，装叉的地方点到为止，我想表达的是，“黑盒子”终结者阿里高层也许是战略思维，就像画一个脑图，不能只有一个root，而是树状的分支直到最细粒度。而工程师思维却往往与此相反。工程师讲求implementation。而implementation，讲究封装，与我不重要之处，我甚至可以剪刀加浆糊的拿来一把。

下面说正题。花了几天研究了下Hexo，不求甚解，只求达成目的，搭个小博客方便写写文章。第一天想的是在linux上搭了然后本地起服务run起来。一切正常，完全没问题。第二天了解的github提供host，而且速度不错。因此不妨在个人mac上搭上环境，必要时deploy一把。

简单说来，mac上搭建Hexo有几个步骤：

  1. 安装node
  2. 安装Hexo： $npm install -g hexo-cli
  3. hexo init …
  4. cd …
  5. npm install

事实上，可能会遇到各种报错：

至于什么错都懒得去细看，大概是权限，被墙，安装不正确之类。下面说说我通过google找到的解决办法(对应上文的几点)：

  1. Try to use Node 0.10.x instead：$nvm install 0.10
  2. 安装Hexo： $sudo cnpm install hexo —no-optional
  3. hexo init …
  4. cd …
  5. npm install 替换成：$sudo cnpm install

以下几篇官方的issue和文章供参考：

  • https://github.com/hexojs/hexo/issues/1055
  • https://github.com/hexojs/hexo/issues/892
  • http://freesouls.github.io/2015/06/06/fix-hexo/

大概就这么几个命令，反复折腾即可:)
